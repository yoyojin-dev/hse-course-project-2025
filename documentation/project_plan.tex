\section{Работа над проектом и распределение задач по участникам}

\subsection{Этапы реализации проекта}

В рамках курсового проекта был применён метод итерационной разработки, предполагающий поэтапное уточнение требований и постепенное наращивание функциональности. Первоначально был выполнен выбор технологического стека и реализован минимальный жизнеспособный продукт (MVP — Minimum Viable Product), содержащий базовый функционал проекта:

\begin{itemize}
	\item Главное меню игры с возможностью сформировать команды и запустить игру;
	\item Командные и общие доски;
	\item Карточки проектов и задач с функционалом блокировки и перетаскивания по доске;
	\item Логика "подбрасывания монетки"\ и хода игрока;
	\item Интерфейс для фасилитатора (ведущего) игры;
	\item Генерация карточек с проектами;
	\item Поддержка одной игровой сессии;
	\item Развёртывание базовой инфраструктуры для функционирования игровой системы.
\end{itemize}


Затем система будет дополнялась расширенными возможностями и оптимизациями, среди которых:

\begin{itemize}
	\item Повышение отказоустойчивости и надёжности системы;
	\item Организация фазы ретроспективного обсуждения между игроками;
	\item Доработка пользовательского интерфейса (UI/UX), улучшение визуального дизайна и анимационных решений.
\end{itemize}

\subsection{Функциональная декомпозиция системы}

Проектируемая система включает следующие основные функциональные составляющие:

\begin{itemize}
\item \textbf{Игровая логика}

\begin{itemize}
\item сценарий проведения игры;
\item этапы игры и переходы между ними;
\item сбор и расчёт игровых метрик (WIP, lead time и др.).
\end{itemize}
\end{itemize}

\begin{itemize}
\item \textbf{Пользовательские роли}

\begin{itemize}
\item ведущий (модерация игры, управление процессом);
\item команда (совместное принятие решений);
\item игрок (участие в игровых активностях);
\item наблюдающий (видит ход игры).
\end{itemize}
\end{itemize}

\begin{itemize}
\item \textbf{Сетевая и многокомандная часть}

\begin{itemize}
\item синхронизация состояния игры между участниками;
\item поддержка нескольких команд в рамках одной сессии.
\end{itemize}
\end{itemize}

\begin{itemize}
\item \textbf{Пользовательский интерфейс}

\begin{itemize}
\item интерфейс ведущего;
\item интерфейс команд и игроков.
\end{itemize}
\end{itemize}

\begin{itemize}
\item \textbf{Хранение и обработка данных}

\begin{itemize}
\item данные игровых сессий;
\item результаты игр и статистика.
\end{itemize}
\end{itemize}

\subsection{Распределение задач по участникам}

До начала реализации проекта командой был сформирован общий подход к организации работ, описанный в главе "4 Процессы разработки". Данный подход был разработан Мостовой Александрой совместно с Сухобоком Тимофеем и определил базовые принципы ведения проекта, последовательность выполнения стори и общую логику взаимодействия между участниками команды.

Обеспечение качества продукта находилось в зоне ответственности Чупрына Матвея. Совместно с Овчаровым Иваном они написали значительный объём unit- и end-to-end тестирования, что позволило своевременно выявлять и устранять ошибки как на уровне отдельных модулей, так и на уровне пользовательских сценариев.

Инфраструктурный контур проекта, включая настройку виртуальных машин, конфигурацию окружений и процесс CI/CD, разрабатывался Чуприным Матвеем и Сухобоком Тимофеем. Подробное описание архитектуры инфраструктуры, а также регламентов управления исходным кодом приведено в главе "7.1 Проектирование инфраструктуры и регламентация процессов управления исходным кодом".

Backend- и Frontend-части проекта создавались совместно большинством участников команды, поскольку проект разрабатывался с нуля и на раннем этапе не предполагал жёсткого разделения на Frontend- и Backend-разработчиков. Вместе с тем можно выделить основных контрибьюторов в отдельные направления. Наибольший вклад в Backend-часть приложения внёс Денисов Николай. Существенный вклад в разработку серверной части также сделали Мостова Александра и Сухобок Тимофей.

Во Frontend-часть приложения, UI/UX-дизайн и ручное тестирование наибольший вклад внесла Бакунц Арина. Дополнительно в разработке клиентской части принимали участие Мостова Александра и Денисов Николай.
