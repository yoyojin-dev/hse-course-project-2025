# Требования к системе

## 1. Назначение системы

Разрабатываемая система предназначена для проведения онлайн-тренингов на основе бизнес-игры «Мультикомандный Featureban», направленной на моделирование межкомандного взаимодействия и выявление проблем коммуникации и координации в процессе совместной работы над проектами.

## 2. Пользовательские роли

В системе предполагается наличие следующих ролей:

* **Ведущий** — управляет игровым процессом, настраивает параметры игры, контролирует ход сессии.
* **Команда** — группа участников, совместно выполняющих игровые задачи.
* **Игрок** — отдельный участник команды, взаимодействующий с системой в рамках игровых правил.

## 3. Функциональные требования

* Создание и управление игровыми сессиями.
* Поддержка нескольких команд в рамках одной игровой сессии.
* Отображение состояния игры в реальном времени для всех участников.
* Сбор и отображение ключевых метрик игрового процесса.
* Возможность конфигурации параметров игры ведущим.

## 4. Нефункциональные требования

* Доступность системы через веб-браузер.
* Поддержка одновременной работы нескольких пользователей.
* Масштабируемость для проведения тренингов с несколькими командами.
* Интуитивно понятный пользовательский интерфейс.

## 5. Ограничения и допущения

* Реализация ориентирована на использование в учебных и тренинговых целях.
* Функциональность системы может быть расширена в дальнейшем по результатам обратной связи.
